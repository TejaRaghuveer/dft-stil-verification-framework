# Week 3 Production Deliverables

I use this folder as the handoff point for everything generated by the Week 3 flow. After each run of `python/integrated_flow/week3_production.py`, this is the first place I check to confirm outputs are complete and ready for review.

| Folder | Contents |
|--------|----------|
| **STIL_Patterns/** | `patterns_generated.stil`, `patterns_compressed_annotated.stil` |
| **Reports/** | Compression, timing at-speed, formal, DRC, STIL validation, HTML/JSON/XML/CSV reports, failure analysis |
| **Artifacts/** | Placeholder for golden responses, netlist pointers, TB configs |
| **Archive/** | Version snapshots (copy tagged release trees here) |

## How to Run It

From repository root (Windows PowerShell):

```powershell
$env:PYTHONPATH="python"
python python/integrated_flow/week3_production.py --atpg patterns/stuck_at.stil --out deliverables --tck-mhz 200
```

With simulation results:

```powershell
python python/integrated_flow/week3_production.py --pattern-db out/integrated/pattern_db.json --results-csv out/integrated/results/full_results.csv --out deliverables
```

Dry-run (minimal placeholder pattern DB):

```powershell
python python/integrated_flow/week3_production.py --dry-run --out deliverables
```

## What I Changed

I structured the outputs into four consistent folders (`STIL_Patterns`, `Reports`, `Artifacts`, `Archive`) so it stays predictable from run to run. I also kept both dry-run and full-result generation on the same entry point, and documented the exact commands I use for local checks and CI so there is less guesswork during bring-up or handoff.

## What Was Tricky

The main challenge was that generated content can differ based on whether real simulation CSV data is available. During review, that made it easy to confuse dry-run outputs with full-result outputs unless the run mode and destination were clearly tracked.

## How I Fixed It

I treated `week3_production.py` as the single source for all deliverable generation and documented both execution paths clearly so expected outputs are explicit before a run starts. I also kept folder naming stable to make validation, review handoff, and archive steps straightforward.
