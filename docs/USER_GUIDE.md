# User Guide - DFT-STIL Verification Framework

## Quick Start

### Prerequisites

1. **Simulator**: One of the following:
   - Synopsys VCS
   - Mentor Graphics QuestaSim
   - Cadence Xcelium

2. **UVM Library**: UVM 1.2 or later

3. **Python**: Python 3.6 or later with required packages:
   ```bash
   pip install pyyaml
   ```

4. **Environment Variables**: Set based on your simulator:
   ```bash
   # For VCS
   export VCS_HOME=/path/to/vcs
   
   # For QuestaSim
   export QUESTA_HOME=/path/to/questa
   
   # For Xcelium
   export CDS_HOME=/path/to/xcelium
   ```

### Running Your First Test

1. **Generate Example STIL File**:
   ```bash
   make stil_example
   ```

2. **Validate Project Structure**:
   ```bash
   make validate
   ```

3. **Compile and Run**:
   ```bash
   make SIMULATOR=vcs TEST_NAME=base_test
   ```

   Or use the simulation script:
   ```bash
   ./scripts/run_simulation.sh -s vcs -t base_test
   ```

## Configuration

### Test Configuration

Edit `config/test_config.json` to customize test parameters:

```json
{
  "test_config": {
    "agents": {
      "jtag": {
        "clock_period_ns": 100,
        "reset_cycles": 10
      }
    },
    "stil": {
      "output_file": "output.stil"
    }
  }
}
```

### UVM Test Configuration

In your test class, override `configure_test()`:

```systemverilog
class my_test extends base_test;
    function void configure_test();
        super.configure_test();
        cfg.jtag_clock_period_ns = 50;  // 20MHz
        cfg.num_test_patterns = 200;
    endfunction
endclass
```

## Creating Custom Tests

### 1. Create a Test Sequence

```systemverilog
class my_sequence extends base_sequence;
    task body();
        jtag_transaction txn;
        repeat(10) begin
            txn = jtag_transaction::type_id::create("txn");
            start_item(txn);
            txn.tms = 0;
            txn.tdi = $urandom();
            finish_item(txn);
        end
    endtask
endclass
```

### 2. Create a Test Case

```systemverilog
class my_test extends base_test;
    function void configure_test();
        super.configure_test();
        cfg.verbosity = UVM_HIGH;
    endfunction
    
    task run_test_sequence();
        my_sequence seq = my_sequence::type_id::create("seq");
        seq.start(env.jtag_agent_inst.sequencer);
    endtask
endclass
```

### 3. Run Your Test

```bash
make TEST_NAME=my_test
```

## STIL Pattern Generation

### Automatic Generation

The STIL subscriber automatically generates STIL files during simulation. Configure output file:

```systemverilog
cfg.stil_output_file = "my_patterns.stil";
cfg.stil_enable_generation = 1;
```

### Manual Generation

Use Python script:

```bash
python python/stil_generator/stil_template_generator.py output.stil
```

### Customizing STIL Output

Modify `stil_subscriber.sv` to customize:
- Signal definitions
- Timing information
- Pattern format
- Procedures

## ATPG Pattern Parsing

### Parse TetraMAX Output

```bash
make parse_atpg ATPG_FILE=patterns.tmax ATPG_FORMAT=tetramax
```

Or directly:

```bash
python python/atpg_parser/atpg_parser.py tetramax patterns.tmax
```

### Supported Formats

- `tetramax`: Synopsys TetraMAX
- `fastscan`: Mentor Graphics FastScan
- `encounter`: Cadence Encounter Test

## Waveform Analysis

## Troubleshooting

### Common Issues

1. **Interface Not Found**:
   - Ensure interfaces are set in config database in `tb_top.sv`

2. **STIL File Not Generated**:
   - Check `cfg.stil_enable_generation = 1`
   - Verify file path is writable

3. **Compilation Errors**:
   - Check UVM library path
   - Verify all SystemVerilog files are included

4. **Simulation Hangs**:
   - Check for missing objections in test
   - Verify clock is running

## Best Practices

1. **Use Configuration Objects**: Don't hardcode parameters
2. **Extend Base Classes**: Maintain framework structure
3. **Add Assertions**: Use SVA for protocol checking
4. **Log Transactions**: Enable transaction logging for debugging
5. **Validate Inputs**: Use validators before simulation
6. **Document Customizations**: Comment all modifications
